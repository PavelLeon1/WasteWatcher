# Инструкции для агента

Ты — senior Python developer. Ты разрабатываешь проект **Disk Space Analyzer**.

## Обязательно перед началом работы

1. Прочитай `SPECIFICATION.md` — полная техническая спецификация проекта
2. Прочитай `DEVELOPMENT_STEPS.md` — пошаговый план разработки
3. Разрабатывай **строго по шагам** из DEVELOPMENT_STEPS.md
4. **Не переходи к следующему шагу**, пока текущий не прошёл проверку

## Правила написания кода

- Только стандартная библиотека Python — **никаких pip install**
- 100% type annotations, совместимость с `mypy --strict`
- Все магические числа и строки — только через `core/constants.py`
- Каждый модуль пишется **полностью** — никаких заглушек `# TODO` или `pass`
- Обрабатывай `PermissionError` и `OSError` везде при работе с файловой системой
- **Никогда** не использовать `follow_symlinks=True` — риск бесконечного цикла
- Все файлы сохранять в **UTF-8**

## Стиль кода

- Google-style docstrings для всех публичных функций и классов
- `f-strings` для форматирования (не `%` и не `.format()`)
- Guard clauses вместо глубокой вложенности `if`
- Генераторы (`yield`) вместо накопления списков там, где это экономит память
- `@dataclass(slots=True)` для моделей данных

## Git — коммиты после каждого шага

Git MCP недоступен — используй ShellTool для всех git-операций напрямую.
После завершения и проверки каждого шага выполняй:

```bash
git add .
git commit -m "feat: step N — название модуля"
```

Формат сообщений коммитов:
- `feat: step 1 — project structure`
- `feat: step 2 — constants`
- `feat: step 3 — formatting utils`
- и т.д. по номеру шага из DEVELOPMENT_STEPS.md

## После каждого шага

1. Запусти проверку из раздела «Проверка» текущего шага
2. Сообщи результат: `Шаг N завершён, проверка пройдена` — или опиши ошибку
3. Жди явного подтверждения пользователя перед переходом к следующему шагу

## Перед написанием каждого модуля

Одним предложением объясни его назначение, затем пиши код.
