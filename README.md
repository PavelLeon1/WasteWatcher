## Описание

**Disk Space Analyzer** — это CLI-инструмент для анализа использования дискового пространства. Он сканирует указанную директорию и все вложенные каталоги, вычисляя для каждого файла специальный **«индекс бесполезности»**, который помогает выявить файлы, занимающие место, но не используемые в течение длительного времени.

### Ключевые возможности

- 🔍 **Рекурсивное сканирование** — обход всех вложенных директорий
- 📊 **Интерактивный HTML-отчёт** — с графиками, фильтрацией и сортировкой
- 🎯 **Умная сортировка** — по индексу бесполезности (размер × время простоя)
- ⚙️ **Гибкая фильтрация** — по размеру, расширению, времени простоя, глубине
- 🚀 **Высокая производительность** — потоковая обработка, O(1) память
- 📦 **Zero Dependencies** — только стандартная библиотека Python

## Использование

### Базовый синтаксис

```bash
python analyzer.py [OPTIONS] PATH
```

### Примеры команд

#### 1. Базовый анализ текущей директории

```bash
python analyzer.py . --output report.html
```

#### 2. Анализ с фильтром по размеру и времени простоя

```bash
python analyzer.py /var/log --min-size 1MB --min-idle 30 --output old_logs.html
```

#### 3. Анализ с исключением директорий

```bash
python analyzer.py /home --exclude /home/.cache /home/.local --depth 4 --verbose
```

#### 4. С экспортом сырых данных в JSON

```bash
python analyzer.py /data --json raw_data.json --output report.html
```

#### 5. Только определённые расширения, топ-100 файлов

```bash
python analyzer.py ~/Downloads --ext .log .tmp .bak --top 100 --verbose
```

### Все опции командной строки

| Опция | Описание | По умолчанию |
|---|---|---|
| `PATH` | Путь к директории для сканирования | **Обязательно** |
| `--output FILE` | Путь к выходному HTML-файлу | `report.html` |
| `--json FILE` | Путь к JSON-файлу с сырыми данными | — |
| `--min-size SIZE` | Минимальный размер (10B, 1KB, 5MB, 2GB) | — |
| `--min-idle DAYS` | Минимальный простой в днях | `0` |
| `--ext EXT [EXT ...]` | Фильтр по расширениям (`.log .tmp`) | — |
| `--exclude DIR [DIR ...]` | Исключить директории | — |
| `--depth INT` | Максимальная глубина обхода | Без ограничений |
| `--top INT` | Показывать топ N файлов (0 = все) | `0` |
| `--no-hidden` | Игнорировать скрытые файлы и папки | `False` |
| `--verbose` | Подробный вывод в консоль | `False` |

## Индекс бесполезности

### Формула

```
uselessness_index = size_bytes × idle_days
```

Где:
- **`size_bytes`** — размер файла в байтах
- **`idle_days`** — количество дней с последнего доступа (`atime`)

### Примеры расчёта

| Файл | Размер | Последний доступ | Idle Days | Индекс |
|---|---|---|---|---|
| `cache.tmp` | 100 MB | 90 дней назад | 90 | **9 000 MB·days** |
| `log.txt` | 10 MB | 30 дней назад | 30 | **300 MB·days** |
| `data.json` | 1 MB | вчера | 1 | **1 MB·days** |

### Интерпретация

- **Высокий индекс** → Файл большой и давно не используется → **Кандидат на удаление**
- **Низкий индекс** → Файл маленький или недавно использовался → **Оставить**

### Отображение

В отчёте индекс отображается в нормализованном виде: **`N.NN MB·days`**

## Известные ограничения

### 1. Символические ссылки

**Не следуют симлинкам** — риск бесконечного цикла. Файлы-симлинки пропускаются.

### 2. Права доступа

Файлы без прав на чтение **логируются и пропускаются**, сканирование продолжается.

### 3. `ctime` на разных ОС

| ОС | `ctime` означает |
|---|---|
| **Windows** | Время создания файла |
| **macOS** | Время изменения метаданных (есть `birthtime`) |
| **Linux** | Время изменения метаданных (inode change time) |

На macOS используется `st_birthtime` для истинной даты создания.

### 4. Длинные пути (Windows)

Windows имеет ограничение в 260 символов для путей. Пути длиннее могут быть обрезаны или пропущены.

### 5. `atime` может быть неточным

На некоторых системах `atime` обновляется не для каждого доступа (например, `relatime` в Linux).
